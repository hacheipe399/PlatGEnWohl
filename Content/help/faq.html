Q: Where are the Level Settings?

A: If you need to change global level settings, open the menubar [Level->Properties]. But if you need to configure the current section, you must open the

Section Properties window in the menu [View -> Section Settings].

Q: Why doesn't music play while editing?
A: To play music, you simply need to press the note "♪" icon on the toolbar, or you can set autostart playing of music:
Open in menubar [Tools -> Application Settings], and select the option, "Autoplay music after opening file", and music will automatically play each time you open the file. If music still doesn't play after this, please, check the data/music directory, it must contain the SMBX music files. If the folder doesn't exist, please copy your music folder from SMBX, or download the music pack from our site.

For Microsoft Windows users: If music stilll doesn't play, follow these steps to add a new system variable:
Simply run "fix_music.bat" from editor folder with administrator rights to automatically set this variable, or you can create it manually:

QT_PLUGIN_PATH

and set its value equal to the full path of your pge_editor's directory (For example: C:\Games\SMBX\PGE_Editor\ )

Q: I can't find "auto-align" option!

A: This option is called "snap to grid" in this editor. To disable/enable it, please, click the sharp icon "#" on the toolbar.

Q: I see ghosties instead NPCs, Why?

A: The "Ghostie" is a dummy image sprite. It appears if you use a broken custom image, an image in a non-supported format, corrupted image, etc.
To fix this error, you must convert the image to a valid format. Supported file formats are: PNG, GIF, JPG, BMP. You can add more supported image formats, if you place special plugins into the imageformats dir, which you can get from the Qt website, here.

Q: Do the graphics use the same format as SMBX?

A: Yes, in the SMBX-compatible configs you can use the graphics format of SMBX:
 - GIF file format without transparency
 - Must use black/white mask to apply transparency
 - Animation frames of sprite going down vertically.
 - Playable character sprite has a 10x10 frame table, and a total image size of 1000x1000 pixels and fixed by the position of the character's hitboxes.
 - All custom SMBX graphics are fully compatible with this engine.

 

Q: Is this a new version of SMBX?

A: This project is offered as a successor of SMBX. It is not SMBX, but fully compatible with it.
SMBX -  is a game, PGE -  is a game engine. All games are created based on their own game engines.
SMBX - closed source, PGE - Libre and Open Source.
This project can be used not only for Mario games based on SMBX, but for others too (while being entirely open source). Because it allows full flexibility and customizing (all items are defined by INI files), and NPCs will be allowed to be programmed by a script language, this allows one to create custom NPCs of their own without being restricted to using the AI of an existing NPC.

 

Q: Why background repeats only horisontal and doesn't repeats vertical?

A: Because you probably are using a single-row background, which is not tiled vertically.

I recommend that you read more about backgrounds here.
